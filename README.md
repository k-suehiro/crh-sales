# 浜松事業所 営業課題ヒアリング

浜松事業所の営業担当者が抱える課題をヒアリングし、スプレッドシートに記録するための Google Apps Script（GAS）Webアプリです。スマートフォンからも入力しやすい4ステップフォームで、1件ずつ登録できます。

## 概要

- **対象**: 浜松事業所の営業担当者（社内ヒアリング）
- **用途**: 営業活動の困りごとを収集・整理し、改善施策の検討材料とする
- **出力先**: 紐づけたスプレッドシートの「課題管理」シート

## 主な機能

- 4ステップの入力フォーム（対象者情報 → 困っていること → 影響の整理 → メモ）
- 社員DBから浜松事業所の社員を検索・選択（漢字・ひらがな・カタカナ対応）
- 担当者選択時に所属部署・担当エリア・役職を自動入力
- メールアドレスを裏側で取得し、スプレッドシートに保存
- 「リスト」シートのマスタからプルダウン選択肢を読み込み

## ファイル構成

| ファイル | 説明 |
|---|---|
| `main.gs` | サーバー側ロジック（社員取得・フォーム送信・シート書き込み） |
| `index.html` | フロントエンド（フォームUI・インクリメンタルサーチ） |
| `appsscript.json` | GAS プロジェクト設定 |
| `.clasp.json` | Clasp 用スクリプトID設定 |

## スプレッドシート構成

### 課題管理シート

1行目に以下のヘッダーを設定してください。

| 列 | ヘッダー名 |
|---|---|
| A | No. |
| B | 登録日時 |
| C | 営業担当者名 |
| D | 所属部署 |
| E | 営業歴（年） |
| F | 役職 |
| G | （未使用） |
| H | 担当エリア |
| I | 課題カテゴリ |
| J | 困っている内容 |
| K | 発生頻度 |
| L | 深刻度 |
| M | 改善したい度合い |
| N | いまの対処方法 |
| O | 月あたり影響時間 |
| P | ツールAI改善余地 |
| Q | 社内打開見込み |
| R | メモ |
| S | ステータス |
| T | メールアドレス |

### リストシート

プルダウン用の選択肢を列ごとに管理します。以下の列名が必要です。

- 課題カテゴリ
- 発生頻度
- 深刻度
- 改善したい度合い
- ツールAI改善余地
- 社内打開見込み
- ステータス

## 社員DB連携

別スプレッドシートの社員DBを参照します。`branch` が「浜松事業所」の行のみを対象とします。

| 列名 | 用途 |
|---|---|
| `branch` | 事業所フィルタ |
| `kanji` | 氏名（検索・表示） |
| `mail` | メールアドレス（非表示で保存） |
| `division` | 所属部署（自動入力） |
| `location` | 担当エリア（自動入力） |
| `position` | 役職（自動入力） |
| `kana` 等 | ひらがな検索用（`hiragana` / `yomi` 等も自動検出） |

社員DBの閲覧権限が、GAS を実行するアカウントに付与されている必要があります。

## セットアップ

### 1. スプレッドシートを用意

- 「課題管理」「リスト」シートを作成
- 上記の列構成・ヘッダーを設定

### 2. GAS プロジェクトと紐づけ

このリポジトリは次のスクリプトIDに紐づいています。

```
1JB37hfEyswDLvXpFD7bLROIbMaKmcQcQkHTRw3zaDPl6hCGA6eTrhPU4
```

スプレッドシートにバインドされた GAS プロジェクトとして利用してください。

**重要（Webアプリ利用時）:** Webアプリからは `getActiveSpreadsheet()` が使えないため、ヒアリング用スプレッドシートIDの登録が必要です。次のいずれかを実施してください。

1. **推奨:** ヒアリング用スプレッドシートを開き、「拡張機能」→「Apps Script」→ 関数 `initializeSpreadsheetId` を選択して **1回だけ実行**
2. **または:** `main.gs` の `HEARING_SPREADSHEET_ID` にスプレッドシートIDを直接記入

スプレッドシートIDは URL の `/d/` と `/edit` の間の文字列です。
### 3. Webアプリとしてデプロイ

1. GAS エディタで「デプロイ」→「新しいデプロイ」
2. 種類: **ウェブアプリ**
3. 実行ユーザー: **デプロイしたユーザー**
4. アクセスできるユーザー: 社内の利用範囲に合わせて設定

### 4. 初回実行時の権限

初回アクセス時に以下の権限承認が必要です。

- スプレッドシートの読み書き
- 社員DB（外部スプレッドシート）の読み取り

## 開発（Clasp）

### 前提

- [Node.js](https://nodejs.org/) がインストール済みであること
- Clasp がインストール・ログイン済みであること

```bash
npm install -g @google/clasp
clasp login
```

### コードの反映

```bash
clasp push
```

### ローカルへ取得

```bash
clasp pull
```

## 入力フロー

1. **対象者情報** — 営業担当者を検索・選択（部署・エリア・役職は自動入力）、営業歴を入力
2. **困っていること** — 課題カテゴリ・内容・現在の対処方法
3. **影響の整理** — 頻度・深刻度・改善意向など
4. **メモ** — 補足情報

登録後、課題管理シートに1行追加されます。
